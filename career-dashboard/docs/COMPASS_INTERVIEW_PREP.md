## The company (verified from public sources)
- Independent **insurance brokerage** (Compass Insurance Advisors, LLC), founded **2014**, HQ **Draper, UT** (38 E Scenic Pointe Dr, Ste 140, 84020; also Minuteman Dr, Draper; plus a South Jordan office). Privately held, **~44–45 employees, growing (+20% YoY)**. Site: insurewithcompass.com.
- **They broker, not underwrite** — compare **100+ carriers** to place clients in the best plan. Lines: **Health, Medicare, Group/employee benefits, Life, Supplemental, Retirement.**
- **Revenue model:** carrier **commissions** (no extra cost to client; health/Medicare premiums are legally fixed). Pitch: *local, personalized, "not a call center."* ACA "Marketplace Circle of Champions" recognition.
- **Why they have software (key for the role):** they give clients **free benefits-administration / online-enrollment software** (employee onboarding, PTO requests, payroll integration, a mobile app to view benefits) plus **plan-comparison tools** agents use. The dev team builds **internal agent tools + client-facing benefits/enrollment web apps.**

## The job (from the live posting)
Duties: build/maintain front-end features for **internal and client-facing** web apps; clean **HTML, CSS/SCSS, JavaScript, Bootstrap**; **React** UIs; integrate **REST APIs** and data from **MySQL, Laravel, Node.js**; **maintain and enhance existing PHP-based applications**; debug UI/UX. Qualifications: *"recent grad or self-taught with strong portfolio"* + basics of HTML/CSS/SCSS/JS/Bootstrap, React, Laravel, PHP & MySQL, Node.js, REST APIs, Git, problem-solving, communication.

## What the role actually is
A **frontend-leaning full-stack maintenance** role — *not* greenfield React. Day-to-day: React + Bootstrap UI for enrollment/benefits/agent portals, wiring to REST APIs, and **upkeep on a legacy PHP/Laravel + MySQL app.** The line *"maintain and enhance existing PHP-based applications"* is the real signal: they want someone willing to learn PHP/Laravel to care for an existing system, not just write new React.

## Two things that matter for Camron
1. **Mentorship is built in.** The team is small (Technical Systems Manager, Technical Systems Assistant, UI/UX Designer, Frontend Developer), and Compass is **also hiring a *senior* Frontend Developer whose posting says "provide code review and mentoring to a junior developer"** — i.e., this role. Strong talking point.
2. **Small company:** broad exposure + visible impact, but **communication and being easy to work with weigh heavily.** Lean on Flores (detail/accuracy) and sales (communication).

## Likely interview themes
- React fundamentals (components, state, props), REST integration, Git — covered by VocalLearn + Career Dashboard.
- Aptitude/willingness to learn **PHP/Laravel/MySQL** (be honest he hasn't shipped PHP; point to fast learning of Rust/TS/Python).
- "Why Compass / why insurance" — mentored first role at a growing local company where he can own real features.
- Behavioral: reliability, communication, ambiguity in a small team.

## Who interviews (expectation)
Likely a **non-technical first screen** (recruiter / COO / ops or the Technical Systems Manager), then a **technical round** with the Technical Systems Manager or the incoming senior Frontend Developer. Prepare **both**: a plain-English version for non-technical folks *and* deeper React/REST/Git answers for the technical interviewer. Don't assume the first interviewer knows the jargon.
