> **Context for you (VocalLearn tutor):** I'm Camron, a self-taught developer preparing for junior software developer interviews — my first target is a Junior Frontend Developer role that uses React, JavaScript, HTML/CSS/Bootstrap, REST APIs, PHP/Laravel, MySQL, and Git. Turn each lesson below into a friendly, analogy-rich course using the production effect: teach a point, then ask me to say it back in my own words, score my recall, and review until I've mastered it. Keep analogies concrete and everyday.
>
> **Lesson 1 — Frontend & React Fundamentals**
> 1. **HTML** — the markup that defines a web page's structure and content (headings, buttons, forms).
> 2. **CSS** — the language that styles HTML (color, layout, spacing, fonts). **SCSS** is an enhanced CSS with variables and nesting.
> 3. **Bootstrap** — a CSS framework of ready-made, responsive components (buttons, grids, navbars) so you don't style everything from scratch.
> 4. **JavaScript** — the language that makes pages interactive: responding to clicks, updating content, and calling servers.
> 5. **React** — a JavaScript library for building user interfaces out of reusable pieces called components.
> 6. **Component** — a self-contained, reusable piece of UI (like a button or form) that you combine to build a page.
> 7. **Props** — the inputs passed *into* a component to configure it (like arguments to a function); read-only, set by the parent.
> 8. **State** — data a component owns and can change over time; when state changes, React re-renders that part of the screen.
> 9. **JSX** — the HTML-like syntax written inside React to describe what the UI should look like.
> 10. **DOM (Document Object Model)** — the browser's live, tree-shaped model of the page that JavaScript reads and updates to change what the user sees.
>
> **Lesson 2 — APIs, Data & Version Control**
> 1. **API** — a defined way for two programs to talk to each other; a contract for requesting and exchanging data.
> 2. **REST API** — a common web-API style where you act on "resources" (like users or jobs) over HTTP using standard actions.
> 3. **HTTP methods** — the verbs of web requests: **GET** (read), **POST** (create), **PUT/PATCH** (update), **DELETE** (remove).
> 4. **Endpoint** — a specific API URL for a resource or action (e.g., `/users/123`).
> 5. **JSON** — the lightweight text format most APIs use to send structured data as key–value pairs.
> 6. **Database / MySQL** — an organized data store; a *relational* database like MySQL keeps data in tables of rows and columns.
> 7. **PHP** — a server-side language used to build web-app backends (the Compass role maintains PHP apps).
> 8. **Laravel** — a popular framework for building web apps in PHP, giving structure for routing, databases, and more.
> 9. **Git** — a version-control tool that tracks changes to code over time so you can review history and collaborate safely.
> 10. **Git workflow (commit / branch / pull request)** — a **commit** is a saved snapshot of changes; a **branch** is a separate line of work; a **pull request** proposes merging a branch's changes after review.
